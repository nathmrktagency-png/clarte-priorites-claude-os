# Clarté & Priorités Marketing — Claude OS

Ce repository contient les éléments non confidentiels du système Claude de Clarté & Priorités Marketing.

## Objectif

Structurer, versionner et améliorer :
- les prompts ;
- les instructions Claude ;
- les procédures internes ;
- les templates anonymisés ;
- les futures Skills ;
- les scripts simples ;
- les règles d’utilisation de Claude Code ;
- les automatisations non sensibles.

## Règle absolue de confidentialité

Ce repository ne doit jamais contenir :
- données clients réelles ;
- contrats signés ;
- devis signés ;
- factures ;
- emails privés ;
- données bancaires ;
- mots de passe ;
- clés API ;
- captures analytics client ;
- documents confidentiels ;
- dossiers clients complets.

Les données clients restent dans Google Drive, dans un dossier et un Project Claude séparés par client.

## Positionnement

Clarté & Priorités Marketing est un cabinet junior structuré de cadrage marketing pour dirigeants de PME locales B2C.

Le business vend :
- clarté ;
- structure ;
- diagnostic ;
- priorisation ;
- cohérence ;
- pilotage ;
- livrables professionnels.

Le business ne vend pas :
- promesse de ROI ;
- garantie de croissance ;
- garantie de ventes ;
- agence 360 degrés ;
- exécution directe sans audit.

## Workflow business

Qualification → Audit payant → Dispositif Simple ou Structuré → Accompagnement éventuel.

Aucun dispositif sans audit préalable.

## Utilisation de Claude Code

Claude Code peut aider à organiser et modifier ce repository, mais il doit toujours :
1. proposer un plan avant modification ;
2. ne jamais supprimer sans validation ;
3. ne jamais ajouter de données sensibles ;
4. travailler sur une branche ou une copie si nécessaire ;
5. demander validation humaine avant commit important.
